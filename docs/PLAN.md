# Escapist — 개인용 면접 준비 앱

## 개요

Claude CLI 세션 기반의 면접 준비 앱. 질문을 등록하고, Claude가 면접관이 되어 답변을 평가하고 꼬리질문으로 깊이를 파고든다.

## 동기

면접에서 "var/let/const 차이", "이벤트루프" 같은 기본 질문에 막혔던 경험에서 출발. 실무만 하다 보면 놓치는 기초 지식을 체계적으로 준비하기 위한 도구.

## 핵심 컨셉

- **Claude CLI를 면접관으로** — API 키 없이 구독만으로 동작
- **꼬리질문 트리** — 하나의 질문에서 여러 방향으로 분기하며 깊이 탐구
- **세션 기반 대화** — `--resume`으로 맥락 유지, 껐다 켜도 이어가기
- **개인 학습 도구** — 나만의 면접 질문 DB를 키워나가는 앱

## 기술 스택

| 영역 | 스택 |
|------|------|
| 프론트엔드 | React + Vite + TypeScript |
| 스타일 | Tailwind CSS |
| 서버 | Node.js + Express + WebSocket |
| AI | Claude CLI (`claude -p`, `--resume`) — 멀티 에이전트 |
| 다이어그램 | Mermaid.js (개념 시각화) + React Flow (꼬리질문 트리) |
| 차트 | Recharts |
| 데이터 | fs 기반 JSON/JSONL |

## 아키텍처

```
[React 프론트엔드] ←WebSocket→ [Node 서버] ←child_process→ [Claude CLI]
                                     ↕
                              [fs 기반 데이터]
                              ├── questions/     (질문 DB)
                              ├── sessions/      (세션 매핑)
                              └── scores/        (점수/평가)
```

## 화면 구조

앱은 크게 **사이드바 + 메인 영역**의 2컬럼 레이아웃.

### 글로벌 레이아웃

```
┌──────────────────────────────────────────────────┐
│  [로고] Escapist                         [설정]   │
├────────────┬─────────────────────────────────────┤
│            │                                     │
│  사이드바   │            메인 영역                 │
│  (240px)   │                                     │
│            │                                     │
├────────────┴─────────────────────────────────────┤
│  (하단 네비게이션 바)                               │
└──────────────────────────────────────────────────┘
```

### 사이드바

모드에 따라 내용이 바뀜.

**학습 모드일 때:**
- "학습 세션" 헤더
- 세션 목록 (최신순)
  - 각 항목: 질문 제목 (1줄 truncate) + 날짜 + 점수 뱃지
  - 활성 세션 하이라이트
- 하단: [+ 새 학습] 버튼

**면접 모드일 때:**
- "면접 기록" 헤더
- 일자별 면접 목록
  - 각 항목: 날짜 + 종합 점수 + 질문 수
- 하단: [오늘의 면접 시작] 버튼

**무한 모드일 때:**
- "무한 모드" 헤더
- 도전 기록 목록
  - 각 항목: 날짜 + 도달 질문 수 + 최종 스트릭
- 최고 기록 하이라이트
- 하단: [도전 시작] 버튼

**대시보드일 때:**
- 사이드바 숨김 또는 축소

### 하단 네비게이션 바

4개 탭:
- **대시보드** (홈 아이콘)
- **학습** (책 아이콘)
- **면접** (마이크 아이콘)
- **무한** (무한대 아이콘)

활성 탭 하이라이트. 각 탭 클릭 시 메인 영역 + 사이드바 전환.

## 화면별 상세 스펙

### 1. 대시보드 (홈)

앱 진입 시 첫 화면. 전체 학습 현황을 한눈에.

**구성요소:**

#### 상단 요약 카드 (가로 나열, 3~4개)
- **총 질문 수**: 등록된 질문 총 개수
- **마스터한 질문**: 상태가 "마스터"인 질문 수 / 전체 (진행 바)
- **평균 점수**: 전체 답변의 평균 점수
- **오늘 학습**: 오늘 답변한 질문 수

#### 약한 질문 목록
- 점수가 낮은 질문 상위 5~10개
- 각 항목: 질문 텍스트 + 카테고리 태그 + 최근 점수 + 시도 횟수
- 클릭 시 해당 질문으로 학습 모드 진입
- 빈 상태: "아직 학습한 질문이 없어요. 학습 모드에서 시작해보세요!"

#### 카테고리별 현황
- 카테고리별로 질문 수 + 평균 점수를 보여주는 차트
- 약한 카테고리 강조

#### 최근 활동
- 최근 학습/면접/무한모드 활동 타임라인 (최근 10건)
- 각 항목: 모드 아이콘 + 질문/면접 제목 + 점수 + 시간

#### 오늘의 면접 히스토리
- 일자별 면접 점수 그래프 (최근 30일)
- 클릭하면 해당 면접 상세로 이동

### 2. 학습 모드

#### 2-1. 새 학습 시작 (질문 선택 화면)

세션이 없거나 [+ 새 학습] 클릭 시.

**구성요소:**
- **검색 바**: 질문 텍스트/태그로 검색
- **필터**: 카테고리 드롭다운, 상태 필터 (전체/미학습/학습중/마스터), 난이도 필터
- **질문 목록**: 카드형 또는 리스트형
  - 각 항목: 질문 텍스트 + 카테고리 + 난이도 별표 + 상태 뱃지 + 최고 점수
  - 클릭 시 학습 세션 시작
- **새 질문 등록 버튼**: 목록 상단 또는 플로팅
- 빈 상태: "등록된 질문이 없어요. 첫 질문을 추가해보세요!"

#### 2-2. 새 질문 등록 모달/페이지

**입력 필드:**
- 질문 텍스트 (필수, textarea)
- 카테고리 (필수, 드롭다운 + 직접 입력)
  - 프리셋: JavaScript, TypeScript, React, CSS, HTML, 네트워크, 브라우저, CS 기초, 자료구조, 알고리즘, 운영체제, 데이터베이스, 기타
- 태그 (선택, 콤마 구분 또는 칩 입력)
- 난이도 (1~5, 별표 또는 슬라이더)
- [등록] / [취소] 버튼

#### 2-3. 학습 세션 (채팅 화면)

학습의 핵심 화면. 채팅 UI 기반.

**상단 헤더:**
- 현재 질문 텍스트 (볼드)
- 카테고리 태그 + 난이도
- [세션 종료] 버튼

**채팅 영역:**
- 면접관(Claude) 메시지: 왼쪽 정렬, 구분 색상
- 내 답변: 오른쪽 정렬
- 평가 카드: 특별한 UI 블록으로 표시
  - 점수 (1~10, 큰 숫자 + 색상: 빨강~초록 그라데이션)
  - 개선점 (불릿 리스트)
  - 모범답안 요약 (접기/펼치기)
- 꼬리질문 선택지: 버튼 형태로 2~3개 표시
  - 클릭하면 해당 꼬리질문으로 대화 계속
- 스크롤은 자동으로 최하단

**하단 입력 영역:**
- 텍스트 입력 (textarea, 여러 줄 가능)
- [제출] 버튼 (Enter로도 가능, Shift+Enter 줄바꿈)
- [모르겠다] 버튼 — 답변 못할 때 사용
- Claude 응답 중일 때: 입력 비활성 + 로딩 인디케이터

**유저 플로우:**
```
질문 선택
  → 면접관: "OOO에 대해 설명해주세요"
  → 사용자: 답변 작성 & 제출
     ├─ [답변 제출] → 면접관: 평가 카드 + 꼬리질문 2~3개
     │   → 꼬리질문 클릭 → 사용자: 답변 → 평가 → 또 꼬리질문... (반복)
     │   → [세션 종료] → 세션 저장 & 질문 점수 업데이트
     └─ [모르겠다] → 면접관: 핵심 개념 설명 + 이해 확인 질문
         → 사용자: 답변 → 평가 → 꼬리질문...
```

**세션 이어가기:**
- 사이드바에서 기존 세션 클릭
- 이전 대화 내역 로딩 (JSONL 파싱해서 채팅 UI에 렌더링)
- 하단 입력으로 이어서 대화 가능 (`--resume`)

### 3. 오늘의 면접

#### 3-1. 면접 설정 화면

[오늘의 면접 시작] 클릭 시.

**설정 옵션:**
- 질문 수: 5 / 10 / 15 / 직접 입력

- 질문 선택 전략: 랜덤 / 약한 질문 우선 / 미학습 우선
- [면접 시작] 버튼

#### 3-2. 면접 진행 화면

**상단:**
- 진행 표시: "3 / 10" (현재 질문 / 전체)
- 프로그레스 바
- 제한시간 설정 시: 타이머 (남은 시간, 30초 이하일 때 빨간색)
- [면접 중단] 버튼

**메인 영역:**
- 면접관 질문 (큰 텍스트)
- 답변 입력 (textarea)
- [제출] / [모르겠다 (패스)] 버튼
- 제출 후:
  - 간단 평가 (점수만 표시, 상세 피드백은 면접 종료 후)
  - 자동으로 다음 질문 전환 (2초 딜레이)

- 꼬리질문 답변 후 다음 메인 질문으로

#### 3-3. 면접 결과 화면

면접 종료 후.

**구성요소:**
- **종합 점수**: 큰 숫자 + 등급 (S/A/B/C/D/F)
- **질문별 결과 리스트**:
  - 질문 텍스트 + 점수 + 상태 (정답/부분정답/오답/패스)
  - 클릭하면 상세 피드백 펼침 (개선점 + 모범답안)
- **카테고리별 분석**: 어떤 카테고리가 약했는지
- **총평**: Claude가 전체적인 피드백 한 문단
- [다시 면접 보기] / [약한 질문 학습하기] / [대시보드로] 버튼

### 4. 무한 모드

#### 4-1. 무한 모드 시작 화면

**표시 내용:**
- 최고 기록: "최대 23문제 연속 정답" (있을 때)
- 규칙 설명:
  - 질문이 랜덤으로 출제됩니다
  - 정답이면 꼬리질문으로 이어집니다
  - 일정 점수 이하면 게임 오버
  - 질문이 모두 소진될 때까지 도전!
- 카테고리 필터 (선택)
- 게임오버 기준 점수: 4점 이하 (설정 가능)
- [도전 시작] 버튼

#### 4-2. 무한 모드 진행 화면

**상단:**
- 현재 스트릭: "🔥 7" (연속 정답 수)
- 총 답변 수: "12문제 완료"
- 남은 질문 수: "38문제 남음"
- 현재 평균 점수

**메인:** 채팅 UI (학습 모드와 동일한 형태)
- 질문 → 답변 → 평가 (점수 즉시 표시)
- 점수가 기준 이상이면: 꼬리질문 자동 진행 또는 다음 질문
- 점수가 기준 이하면: 게임오버 화면으로 전환

**전환 애니메이션:**
- 성공 시: 스트릭 카운터 올라가는 효과
- 실패 시: 화면 흔들림 또는 페이드아웃

#### 4-3. 무한 모드 결과 화면 (게임오버)

**구성요소:**
- **GAME OVER** 텍스트
- 최종 스트릭 / 총 답변 수 / 평균 점수
- 최고 기록 갱신 시: "NEW RECORD!" 표시
- 틀린 마지막 질문 + 모범답안
- 전체 답변 히스토리 (접기/펼치기)
- [다시 도전] / [대시보드로] 버튼

## 컴포넌트 목록

### 레이아웃
- `AppLayout` — 전체 레이아웃 (사이드바 + 메인 + 하단 네비)
- `Sidebar` — 사이드바 컨테이너 (모드별 내용 전환)
- `BottomNav` — 하단 네비게이션 바
- `Header` — 상단 헤더 (로고 + 설정)

### 채팅
- `ChatContainer` — 채팅 메시지 영역
- `ChatBubble` — 개별 메시지 버블 (면접관/사용자 구분)
- `EvaluationCard` — 평가 결과 카드 (점수 + 개선점 + 모범답안)
- `FollowUpButtons` — 꼬리질문 선택 버튼 그룹
- `ChatInput` — 답변 입력 영역 (textarea + 제출 + 모르겠다)
- `LoadingIndicator` — Claude 응답 대기 표시

### 질문
- `QuestionList` — 질문 목록 (검색/필터 포함)
- `QuestionCard` — 질문 카드 (목록 아이템)
- `QuestionForm` — 질문 등록/수정 폼
- `CategoryTag` — 카테고리 태그 칩
- `DifficultyStars` — 난이도 별표

### 면접
- `InterviewSetup` — 면접 설정 화면
- `InterviewProgress` — 진행 표시 (N/M + 프로그레스 바)
- `Timer` — 제한시간 타이머
- `InterviewResult` — 면접 결과 화면
- `GradeDisplay` — 등급 표시 (S/A/B/C/D/F)

### 무한 모드
- `EndlessStart` — 시작 화면 (최고 기록 + 설정)
- `StreakCounter` — 스트릭 카운터 (애니메이션)
- `GameOverScreen` — 게임오버 결과 화면

### 다이어그램 & 트리
- `MermaidRenderer` — Mermaid 코드 블록 → 다이어그램 렌더링
- `FollowUpTree` — React Flow 기반 꼬리질문 트리 시각화
- `TreeNode` — 트리 노드 (질문 + 점수)

### 에이전트
- `AgentSelector` — 에이전트 선택 드롭다운 (면접관/튜터/리서처/다이어그래머)
- `AgentBadge` — 메시지 버블의 에이전트 표시 (아이콘 + 이름)

### 대시보드

## 상태 & 에지케이스

### 로딩 상태
- **Claude 응답 대기**: 채팅 영역에 "면접관이 평가 중..." + 타이핑 애니메이션
- **세션 로딩**: 사이드바에서 세션 클릭 시 이전 대화 불러오는 동안 스켈레톤 UI
- **데이터 로딩**: 대시보드/질문 목록 초기 로딩 시 스켈레톤

### 빈 상태
- **질문 없음**: "등록된 질문이 없어요. 첫 질문을 추가해보세요!" + CTA 버튼
- **세션 없음**: "아직 학습 기록이 없어요. 질문을 선택해서 시작해보세요!"
- **면접 기록 없음**: "면접 기록이 없어요. 오늘의 면접을 시작해보세요!"
- **대시보드 데이터 없음**: 카드에 "---" 표시, 차트 영역에 안내 문구

### 에러 상태
- **Claude CLI 실행 실패**: "Claude CLI에 연결할 수 없어요. CLI가 설치되어 있는지 확인해주세요." + 재시도 버튼
- **Claude 응답 타임아웃**: "응답 시간이 초과됐어요." + 재시도/건너뛰기 버튼
- **세션 resume 실패**: "이전 세션을 불러올 수 없어요. 새 세션으로 시작할까요?" + 새 세션/재시도
- **파일 시스템 에러**: "데이터를 저장/불러올 수 없어요." + 경로 확인 안내
- **WebSocket 끊김**: 상단에 "연결이 끊어졌어요" 배너 + 자동 재연결 시도

### 인터랙션 에지케이스
- **빈 답변 제출**: 제출 버튼 비활성 (입력 없으면)
- **면접 중 브라우저 닫기**: 다음 접속 시 "진행 중인 면접이 있어요. 이어서 할까요?" 확인
- **무한 모드 중 질문 소진**: "모든 질문을 완주했어요! 축하합니다!" + 결과 화면
- **동시에 여러 세션**: 허용하지 않음, 기존 세션 종료 확인
- **제한시간 초과 (면접 모드)**: 자동으로 "시간 초과" 처리, 0점 기록, 다음 질문으로

## 멀티 에이전트

Claude CLI의 `--system-prompt`와 `--allowedTools`로 역할별 에이전트를 구성.

### 에이전트 목록

| 에이전트 | 역할 | CLI 옵션 | 사용 시점 |
|---------|------|---------|----------|
| **면접관** | 질문 출제, 답변 평가, 꼬리질문 생성 | 기본 (도구 없음) | 학습/면접/무한 모드 |
| **CS 튜터** | 개념 설명, 배경 지식 보충, 이해 확인 | 기본 (도구 없음) | "모르겠다" 선택 시 / 사용자가 직접 호출 |
| **리서처** | 공식문서 검색, 최신 스펙/MDN 참조 | `--allowedTools "WebSearch WebFetch"` | 사용자가 직접 호출 / 튜터가 추천 |
| **다이어그래머** | 개념 시각화 (이벤트루프, 프로토타입 체인 등) | 기본 (도구 없음), Mermaid 문법 출력 | 사용자가 직접 호출 / 튜터가 추천 |

### 에이전트 전환 UX

- 채팅 입력 영역 옆에 에이전트 선택 드롭다운 (기본: 면접관)
- 또는 채팅 내에서 `@튜터`, `@리서처`, `@다이어그램` 멘션으로 전환
- 에이전트별 메시지 버블 색상/아이콘 구분
- 같은 세션(`--resume`) 안에서 에이전트만 교체 (시스템 프롬프트 변경)

### 다이어그래머 출력 처리

- Claude가 Mermaid 문법으로 다이어그램 출력
- 프론트에서 Mermaid.js로 렌더링 (코드 블록 감지 → 자동 변환)
- 렌더링된 다이어그램은 채팅 버블 내 인라인 표시

### 꼬리질문 트리 시각화 (React Flow)

- 학습 세션의 꼬리질문 분기를 인터랙티브 트리로 시각화
- 노드: 질문 텍스트 + 점수 뱃지
- 엣지: 분기 관계
- 노드 클릭 시 해당 질문의 답변/평가 상세 표시
- 세션 상세 페이지 또는 사이드 패널에 표시

## 하네스 (시스템 프롬프트)

Claude CLI에 전달할 에이전트별 역할 프롬프트. 모드와 에이전트 조합으로 분기.

### 공통 평가 기준
- 정확성 (핵심 개념을 정확히 설명했는가)
- 깊이 (표면적 답변인가, 원리까지 이해하고 있는가)
- 실무 연관성 (실제 코딩에서 어떻게 적용되는지 언급했는가)
- 설명력 (면접관이 이해하기 쉽게 구조적으로 설명했는가)

### 응답 포맷
Claude는 JSON으로 구조화된 응답을 반환:
```json
{
  "type": "evaluation",
  "score": 7,
  "feedback": "호이스팅 개념은 정확하게 설명했지만...",
  "improvements": ["TDZ에 대한 설명 보충", "실무 예시 추가"],
  "modelAnswer": "var, let, const의 핵심 차이는...",
  "followUpQuestions": [
    "그렇다면 TDZ는 왜 존재하나요?",
    "호이스팅이 실제로 문제가 되는 경우를 예시로 들어주세요"
  ]
```

## 배포 전략

Claude CLI가 서버에서 돌아야 하므로 클라우드 배포 불가. 홈 서버(맥북 에어) 기반.

```
모바일/외부 → Cloudflare Tunnel → 맥북 에어 → Express(:8888) + 빌드된 정적 파일
                                                    ↕
                                              Claude CLI + SQLite
```

- `vite build` → Express에서 정적 파일 서빙 (Vite dev 서버 불필요)
- PM2로 프로세스 관리 (crash 시 자동 재시작, 재부팅 시 자동 시작)
- `git pull && npm run build && pm2 restart`로 배포
- 맥북 에어 뚜껑 닫아도 전원 연결 + 잠자기 방지 설정
